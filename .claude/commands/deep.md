以下のタスクを、品質ループで実行してください。

タスク: $ARGUMENTS

## 手順（この順番を厳守）

1. **作成**: Agent ツールで `sonnet-worker` を起動し（`run_in_background: false` で同期実行）、上記タスクをそのまま渡して成果物を作らせる。

2. **レビュー（1周目）**: Agent ツールで `sonnet-reviewer` を起動し、「元のタスク内容」と「作業担当の報告（成果物の場所・設計判断・自信がない点）」を渡してレビューさせる。

3. **判定**:
   - `APPROVE` → 手順6へ。
   - `REVISE` → 手順4へ。

4. **修正**: SendMessage で **同じ** sonnet-worker（コンテキストを保持している）にレビュー指摘を送り、修正させる。新しいエージェントを起動し直さないこと。

5. **レビュー（2周目）**: 修正結果を SendMessage で同じ sonnet-reviewer に送り、再レビューさせる。
   - `APPROVE` → 手順6へ。
   - まだ `REVISE` → **ここでループを打ち切る**（3周目はしない）。残った指摘は手順6の報告に含める。

6. **最終報告**: ユーザーに以下を報告する：
   - 成果物（変更ファイル一覧、または本文）
   - レビューで見つかり修正された問題点（ループの効果が見えるように）
   - 未解決の指摘があればその内容と、作業担当側の言い分
   - コード変更の場合、コミットするかどうかはユーザーに確認する

## 注意

- あなた（メインエージェント）は自分で成果物を作らない。オーケストレーションに徹する。
- 作業担当とレビュー担当の間で意見が割れたら、両方の言い分を最終報告に載せ、あなたの判断を添える。
- タスクが「単純すぎてレビュー不要」だと感じても、このコマンドが使われた以上ループは実行する。
