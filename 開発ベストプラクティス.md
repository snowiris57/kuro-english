## 1. 環境設計

### 原則：アプリごとに venv で分離する

複数のアプリが同じ Python 環境を共有すると、片方のインストール・更新がもう片方を壊す。
（実例：OpenMythos が torch を 2.5.1 に書き換えて文字起こしアプリが壊れた）

```bat
REM プロジェクト直下に venv を作る
python -m venv .venv

REM 起動.bat では venv の Python を使う
".venv\Scripts\pythonw.exe" "%~dp0main.py"

REM インストール.bat では venv の pip を使う
".venv\Scripts\pip.exe" install パッケージ名
```

### torch を使うアプリは特に要注意

| アプリ | torch 使用 | venv 推奨度 |
|---|---|---|
| 文字起こしアプリ | ✅ | 必須 |
| AI系ツール全般 | ✅ | 必須 |
| Web スクレイピング等 | ❌ | 任意 |

## 2. PyTorch インストール（GPU 自動判定）

### 原則：nvidia-smi で GPU を判定して適切な版を入れる

```bat
nvidia-smi >nul 2>&1
if %errorlevel% == 0 (
    echo NVIDIA GPU 検出 - CUDA 12.1 版をインストール
    pip install torch torchaudio --index-url https://download.pytorch.org/whl/cu121
) else (
    echo GPU なし - CPU 版をインストール
    pip install torch torchaudio --index-url https://download.pytorch.org/whl/cpu
)
```

### torch / torchvision / torchaudio はセットで揃える

バージョンが 1 つでもずれると循環インポートエラーが起きる。
個別に入れず、常に同じインデックス URL から同時にインストールする。

```bat
REM NG：バラバラに入れる
pip install torch
pip install torchvision

REM OK：同じ URL から一括で入れる
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
```

### GPU 検出ロジックはコード側でも防御する

```python
def detect_device():
    try:
        import ctranslate2
        if ctranslate2.get_cuda_device_count() > 0:
            import ctypes
            ctypes.CDLL("cublas64_12.dll")  # DLL が実際にロードできるか確認
            return "cuda", "float16"
    except Exception:
        pass
    return "cpu", "int8"
```

`cuda` デバイスが見えていても DLL がない状態でクラッシュするケースがある。
必ず実際にロードを試みてから返す。

## 3. Windows UI の注意点

### フォルダ選択ダイアログ：initialdir を必ず指定する

```python
# NG：initialdir なし → デスクトップ等の仮想フォルダを COM 経由で開こうとしてエラー
path = filedialog.askdirectory(title="フォルダを選択")

# OK：実パスを渡す
initial = self.output_dir.get() or str(Path.home())
path = filedialog.askdirectory(title="フォルダを選択", initialdir=initial)
```

### ドラッグ＆ドロップ：tkinterdnd2 + windnd の二重構成にする

`tkinterdnd2` の OLE DnD は環境によって動かないことがある。
`windnd`（WM_DROPFILES）をフォールバックとして必ず追加する。

```python
# インポート
try:
    from tkinterdnd2 import TkinterDnD, DND_FILES
    _DND_AVAILABLE = True
except ImportError:
    _DND_AVAILABLE = False

try:
    import windnd
    _WINDND_AVAILABLE = True
except ImportError:
    _WINDND_AVAILABLE = False

# __init__ 内でフック
if _WINDND_AVAILABLE:
    windnd.hook_dropfiles(self.root, func=self._on_windnd_drop)

# windnd コールバック
def _on_windnd_drop(self, paths):
    if not paths:
        return
    raw = paths[0]
    path = raw.decode("utf-8", errors="replace") if isinstance(raw, bytes) else raw
    self.audio_path.set(path)
```

## 4. インストーラー（インストール.bat）テンプレート

```bat
@echo off
chcp 65001 > nul
echo =========================================
echo  Setup: アプリ名
echo =========================================
echo.

REM Python 確認
python --version > nul 2>&1
if %errorlevel% neq 0 (
    echo [ERROR] Python is not installed.
    pause
    exit /b 1
)

REM 通常パッケージ
echo [1/3] Installing packages...
pip install パッケージA パッケージB windnd
if %errorlevel% neq 0 ( pause & exit /b 1 )

REM PyTorch（GPU 自動判定）※ torch を使う場合のみ
echo [2/3] Installing PyTorch...
nvidia-smi >nul 2>&1
if %errorlevel% == 0 (
    echo   GPU detected - CUDA 12.1
    pip install torch torchaudio --index-url https://download.pytorch.org/whl/cu121
) else (
    echo   No GPU - CPU only
    pip install torch torchaudio --index-url https://download.pytorch.org/whl/cpu
)

REM 動作確認
echo [3/3] Checking...
python -c "import torch; print('torch:', torch.__version__, '/ CUDA:', torch.cuda.is_available())"

echo.
echo Done!
pause
```

## 5. よくあるエラーと対処

| エラー | 原因 | 対処 |
|---|---|---|
| `partially initialized module 'torchvision' has no attribute 'extension'` | torch と torchvision のバージョン不一致 | 同じ URL から一括再インストール |
| `Library cublas64_12.dll is not found` | CUDA 版 torch が必要な環境で DLL がない | GPU 判定ロジックを CPU にフォールバック |
| `モニカーに対するオブジェクトがありません` | filedialog に initialdir なし | `initialdir=str(Path.home())` を追加 |
| D&D で赤丸斜線が出る | tkinterdnd2 の OLE DnD が環境依存で失敗 | windnd をフォールバックとして追加 |
| pyannote.audio が torch バージョン不一致警告 | 他アプリが torch を書き換えた | venv で環境分離 |

## 6. バッチファイルの注意点

### 必ず `cd /d "%~dp0"` を入れる（毎回やらかしがち）

`.bat` をダブルクリックしたとき、カレントディレクトリはデスクトップや `C:\Windows\System32` になる場合がある。

```bat
@echo off
cd /d "%~dp0"    ← これを先頭に必ず入れる
python main.py
```

**実例：** PDF解除ツールの初回起動で `python: can't open file 'C:\Windows\System32\pdf_unlocker.py'` エラーが発生。

---

### PowerShell 呼び出し時のクォート規則

バッチから `-Command` で PowerShell を呼ぶとき、シングルクォート内の `$env:TEMP` は**展開されない**（リテラル文字列になる）。

```bat
REM NG：シングルクォートで囲む → $env:TEMP が展開されない
powershell -Command "Invoke-WebRequest -OutFile '$env:TEMP\setup.exe'"

REM OK：Join-Path を使う（クォートなしで変数が展開される）
powershell -Command "$tmp = Join-Path $env:TEMP 'setup.exe'; Invoke-WebRequest -OutFile $tmp"
```

---

### winget はインストール済みでもエラーコードを返すことがある

`winget install` は既インストールのとき `0` 以外を返す環境がある。
**先にファイル存在チェックして winget をスキップする**のが確実。

```bat
if exist "C:\Program Files\Tesseract-OCR\tesseract.exe" (
    echo Already installed. Skipping.
    goto :next
)
winget install --id UB-Mannheim.TesseractOCR --silent ...
```

## 7. 外部 CLI ツールの依存関係は事前に全部調べる

ライブラリが内部で呼ぶ CLI ツールを見落としがち。採用前に公式ドキュメントの「Requirements」を確認する。

### ocrmypdf の依存関係（実例）

| 必要なもの | 役割 | 見落としやすさ |
|---|---|---|
| Tesseract OCR | 文字認識エンジン | 低（主役なので気づく） |
| Ghostscript (`gswin64c`) | PDF 最適化・後処理 | **高（見落としやすい）** |
| 言語パック（`.traineddata`）| 各言語の認識モデル | 中 |

**実例：** Tesseract だけインストールして実行したら `Could not find program 'gswin64c' on the PATH` で大量エラー。

### 外部ツールの PATH 自動検出パターン

ユーザーが PATH を設定し忘れても動くように、コード側でデフォルトインストール先を自動追加する。

```python
import glob, os

def _ensure_ghostscript_in_path():
    for pattern in [r"C:\Program Files\gs\gs*\bin"]:
        matches = glob.glob(pattern)
        if matches:
            latest = sorted(matches)[-1]
            if latest not in os.environ.get("PATH", ""):
                os.environ["PATH"] = latest + os.pathsep + os.environ["PATH"]
            break
```

## 8. データ構造の順序はコメントで明記する

タプルリストのキー・値の順序を間違えると、全体が逆になって気づきにくいバグになる。

```python
# NG：順序が不明確
LANG_OPTIONS = [("日本語", "jpn"), ("英語", "eng")]
for code, label in LANG_OPTIONS:  # 実は逆！ code="日本語", label="jpn" になる
    ...

# OK：(コード, 表示名) と明記 + 変数名を合わせる
LANG_OPTIONS = [  # (tessaract_code, display_name)
    ("jpn",     "日本語"),
    ("eng",     "英語"),
    ("chi_sim", "中国語（簡体字）"),
]
for code, label in LANG_OPTIONS:  # code="jpn", label="日本語" → 正しい
    ...
```

**実例：** PDF文字起こしツールで `LANG_OPTIONS` のタプル順序を逆に定義してしまい、チェックボックスに `jpn` `eng` と表示されて全部 disabled になった。

## 9. 新規ツール開発の標準フロー

1. **GitHub 調査**：既存ツールを検索して使えるか判断する（自作コストと比較）
2. **依存関係の全洗い出し**：ライブラリが必要とする外部 CLI ツールを公式ドキュメントで確認
3. **ファイル構成を決める**：`メインスクリプト / セットアップ.bat / 起動.bat / デバッグ起動.bat`
4. **バッチファイルの基本形**：`cd /d "%~dp0"` + 存在チェック + winget + フォールバック DL
5. **GUIの依存チェック**：起動時に全依存（外部 CLI 含む）を検査してユーザーに案内
6. **出力は元ファイルを上書きしない**：`_unlocked.pdf` `_ocr.pdf` のようにサフィックスを付ける
